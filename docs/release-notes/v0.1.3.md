# v0.1.3

## User-Facing Changes

- Adds `ros2docker init` to scaffold a starter workspace with `ros2docker.json`, `ws/src/`, `catmux.yaml`, and optional `.devcontainer/devcontainer.json`.
- Splits the packaged Docker setup into composable profiles: base `minimal` and `desktop` profiles plus `foxglove`, `zenoh`, `mcap`, and `novatel` add-ons.
- Adds shell completion generation for Bash and Zsh with dynamic completion for profiles, supported ROS distros, and path-taking options.
- Improves host path handling so relative bind mounts resolve from the config file directory and mounted files remain editable from both host and container without sudo.
- Adds a contributor-facing quality model, release runbook, and CI documentation so release and maintenance workflows are explicit.

## Compatibility And Migration

- Breaking changes: none.
- Migration required: none for existing configs.
- The historical full image composition is available as the `project-develnor` convenience profile; new configs can compose smaller profiles explicitly.
- Add-on apt package names currently target the `lyrical` distro. Non-lyrical base image overrides should avoid lyrical-specific add-ons unless the package set is adjusted.

## Docker And Dependency Updates

- Updates the default `osrf/ros:lyrical-desktop-full-resolute` base image digest to `sha256:8c6f54903768a8e344f17385b9323e2a68fe24cf113b67352f66a3faf84c4412`.
- Updates the pinned MCAP CLI from `0.0.62` to `0.2.0` with checksum `53274b6ca922e2078daa02ae32aed75da046f78d6c3da9dc19065254be24b483`.
- Verifies the pinned Zenoh router and `zenoh-plugin-ros2dds` versions remain current at `1.9.0` with unchanged upstream checksums.
- Bumps development tooling and CI dependencies, including Ruff and `actions/checkout`.

## Fixes And Internal Quality

- Hardens release publishing so tag-specific release notes are verified before PyPI publishing can run.
- Adds CODEOWNERS, actionlint, hadolint, shellcheck, dependency-review license gates, and contract tests for CI/docs/config drift.
- Expands unit, contract, and E2E coverage for command rendering, diagnostics, initialization, completions, profiles, Docker build args, and mounted-file ownership.
- Raises the coverage floor to 80 and separates interpreter-independent checks from the Python version matrix.
- Adds scheduled/nightly Docker E2E coverage for the minimal profile, GUI forwarding, SSH-agent forwarding, and full slow release validation.
