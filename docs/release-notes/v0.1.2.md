# v0.1.2

## User-Facing Changes

- Adds explicit Docker run interactivity config keys: `tty` and `stdin_open`.
- Defaults `tty` and `stdin_open` from `run_type`: `bash` and `catmux` are interactive; `command` and `up` are not.
- Clarifies `run_type` behavior: `command` is one-shot, `up` is long-lived, and `ros2docker exec` runs commands inside an already-running container.
- Documents `mount_ws` as the canonical way to mount a config-adjacent `ws` directory into `/ws`.

## Compatibility And Migration

- Breaking changes: none.
- Migration required: none. Existing configs keep their previous interactivity behavior unless they set `tty` or `stdin_open` explicitly.
- Config validation now accepts `tty` and `stdin_open` as public boolean keys and still rejects unknown top-level keys.

## Docker And Dependency Updates

- Keeps package versions tag-derived through `setuptools-scm`; no source version string is added.
- Includes maintenance updates since `v0.1.1`, including typed package metadata, runtime/default coverage, release workflow documentation, issue templates, dependency automation, and CI/coverage gate updates.

## Fixes And Internal Quality

- Adds unit and contract coverage for interactivity defaults, explicit overrides, schema validation, and Docker command rendering.
- Preserves `ros2docker exec` behavior while documenting when to use it.
